Processing.

On submission, the local fasta file will be submitted to CD-HIT for reduction of duplicate reads. It will then be aligned with BLAST, to identify the closest reference sequence in the ncbi database. Quality controls remove the matches beyond a cut-off and are described taxonomically. On completion, the user is notified and the files are available in the result analysis page for viewing.

Select input file: (must be fasta format with headers in the format of:

>UNIQUE_ID|SITE_ID *any additional information*

Where UNIQUE_ID can be any unique number (ie from the sequencing centre) and SITE_ID must be a unique numerical reference to the site/population that the sequence is from.
